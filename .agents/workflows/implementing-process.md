### Objective
- Create and Modify expert level solutions that is not only functional but also adheres to the best practices in performance, security, and maintainability. 

### Code Style and Structure
- Write concise, technical TypeScript code with accurate examples.
- Use functional and declarative programming patterns; avoid classes.
- Favor iteration and modularization over code duplication


## Implementation Process 
1. **Grounding**: Do a quick package.json and online web search about the changes and tools to understand technologies you work with
2. **Deep Dive Analysis**: Begin by conducting a thorough analysis of the task at hand,Approach the problem with analytical rigor. 
3. **Implementation**: Break down the requirements into smaller, manageable parts and thoroughly  consider each step before implementation.
4. **Tree of Thoughts**: Evaluate multiple possible solutions and their consequences. Use subagents approach to explore different paths and select the optimal one.
3. **Review and Optimize**: Perform a self-review of your changes and the code connected to it, looking for areas of potential optimization and improvement.
3. **Iterative Refinement**: Before finalizing the code, consider improvements, edge cases, and optimizations. Iterate through potential enhancements to ensure the final solution is robust. Clean up comments

### UI and Styling
- Use reusable modern UI frameworks (e.g., Tailwind CSS, Shadcn UI, Radix UI) for styling.
- Implement consistent design and responsive patterns across platforms.
- use global theme variants when available
